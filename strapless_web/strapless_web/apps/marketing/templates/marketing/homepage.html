Strapless Dev Style Guide and other stuff

Elements

Elements are like components, but have multiple possible visual states. (Like... elements do...)

They're self-contained in that they don't refer to variables in other files aside from the standard Strapless variables.

They should have the following, in order:

  • HTML classes that the core mixin is written to.
  • Variables (if they have them).
  • A base class to extend (if it has one).
  • A core mixin (with guards if necessary) that accepts a color argument (or two).

Elements shouldn't be too magical, but they can take liberties in contrast/color adjustments.

Color and contrast

Strapless elements don't generally need to be written to work with any color. But ours do because they're the base elements.

We also have to make sure everything stays readable. Use contrast mixins to color text.

Style guidelines

  • Use semicolons between mixin arguments (exceptions allowed for readability)
  • Comments are in lower case. Just like CSS!
  • Rough order of things in an element: variables, generator mixins (and anything else that writes CSS, including written CSS), other mixins, base classes
  • Pollute the global namespace as much as you want, variable-wise. Strapless is a bootstrap-inspired first-files-of-the-project kinda thing. Go crazy. Also, LESS has lazy loading of variables so collisions are usually easy to deal with. The only thing that really needs to be namespaced are elements, so we don't have 8 million .buttons to deal with. Leave "base" out if you can. It's not @padding-base, just @padding.
  • We do use "base" at the end of the base classes for elements. Variables are the wild west. Elements are little more locked down.
  • Want a @margin global variable? Define it. Can't think of a good default value? Set it to @padding. We'll end up with some useful defaults only after experimenting a bit.
  • Strapless occasionally relies on prepositions in mixin names to make mixins easier to read. Take a look at .contrast-text-against() and .make-sl-buttons-for() for examples.
