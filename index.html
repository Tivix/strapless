FAQ

Where are all the classes of things I have to memorize?

There aren't any. The only classes Strapless uses are color names.


Do you have a table with borders, but like, just in between the columns, and not around the whole table?

Go away.


Where's the grid?

There isn't one. Some elements (navbar, footer) try to be smart about where to put stuff. If you need to do more, then grab something that does more and use it on top of Strapless. We recommend: X, Y and Z


Do I have to use the elements? I just want colors.

No, you can comment out elements.less and everything will work fine.


Aren't all these patterns going to bloat my CSS?

Patterns are written as mixins, so their corresponding files aren't included in your CSS unless you use them.


Why didn't you just use CSS for these patterns?

SVG offers more flexibility.


What is elements.less?

It's a base file with some simple, opinionated defaults for all of the standard HTML elements.

Can I add my own SVGs using this pattern?

You can! But it's kind of a pain. We recommend using the library patterns for development feel/texture, and then replacing them later on when your colors are sorted out.


How about a Sass version?

Never. LESS is better. We know which one is more popular and we think everyone is wrong.

(If you're interested in putting together a Sass version, instead of just asking for one incessantly, please get in touch!)


Who is this even for?

Strapless is for developers who have gotten comfortable with frameworks like Bootstrap and Zurb and want something more lightweight for their next project.


You couldn't think up something better than "color2"?

The colors have simple names on purpose. Feel free to reassign any schemeless color to a variable with a better name.
